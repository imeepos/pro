使用说明

任务执行流程：

  1. 历史数据回溯：系统将从起始日期开始，智能拆分时间段抓取历史微博数据
  2. 增量更新：完成历史回溯后，按照设定的间隔定期抓取新的微博数据
  3. 智能拆分：自动根据数据量调整时间段，确保抓取完整不遗漏
  4. 状态监控：实时监控任务状态，支持暂停、恢复和立即执行

注意事项：

  • 起始日期越早，历史数据回溯时间越长
  • 建议首次创建时选择较近的日期进行测试
  • 抓取间隔不宜过短，避免对微博服务器造成压力
  • 账号轮换可以提高抓取成功率，建议开启
